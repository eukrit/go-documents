Signature Service — GO Submissions Workflow

Material & drawing approval sign-off via Google Drive eSignature + bilingual deemed-accepted clause + Slack interactivity routing

The problem

Consultants and clients sit on material/drawing submissions. Without a contractual response window, the contractor's schedule slips while waiting for approval. We needed (1) a low-friction signature path that doesn't require leaving Google Workspace, and (2) a contractual fallback for when the customer simply doesn't respond.

The "Deemed Accepted" clause (bilingual)

Appears verbatim on every submission PDF AND in every submission email body. Single source of truth: go-documents/src/submission_clause.py. Default window 7 working days, configurable via SUBMISSION_RESPONSE_DAYS env var.

EN: Please review and return this submission with written comments within 7 working days of the date above. If no written comments or rejection are received within that period, this submission shall be deemed accepted in full and the contractor will proceed accordingly.
TH: กรุณาตรวจสอบและส่งคืนเอกสารฉบับนี้พร้อมความคิดเห็นเป็นลายลักษณ์อักษรภายใน 7 วันทำการนับจากวันที่ระบุข้างต้น หากไม่ได้รับความคิดเห็นหรือการปฏิเสธเป็นลายลักษณ์อักษรภายในระยะเวลาดังกล่าว ให้ถือว่าเอกสารนี้ได้รับการอนุมัติโดยสมบูรณ์ และผู้รับเหมาจะดำเนินการตามที่เสนอต่อไป

Signature service choice

We use Google Workspace native eSignature (Drive + Docs, GA April 2024 for PDFs). It's bundled with our Business Standard plan, no third-party data sharing, and the signed PDF returns to Drive automatically with an audit certificate. Trade-off: there is no public API for "send for signature" yet (as of 2026-04), so step 5 is a manual UI click. Cloud Run mirrors the PDF to Drive automatically so the sign-out is one click away.

Slack interactivity wiring

Submission Slack cards ship with Approve / Reject / Comment / Open / Drive (Sign) buttons. Each carries an action_id prefixed submission_*. Slack delivers button clicks to the central slack-router Cloud Run service, which forwards by prefix match to go-documents/slack/interactivity.

action_id prefixForwarded toEffect
submission_approve_*go-documents /slack/interactivityStatus → approved, Pub/Sub event, Slack confirms
submission_reject_*go-documents /slack/interactivityStatus → rejected, Pub/Sub event, Slack confirms
submission_comment_*go-documents /slack/interactivityModal opens for reviewer remarks
submission_open_*(URL button)Opens dashboard at /submissions/<sid>
submission_drive_*(URL button)Opens the Drive PDF for manual "Request signature"

Configuration

  • envSUBMISSION_RESPONSE_DAYS — response window (default 7 working days)
  • envSUBMISSIONS_DRIVE_FOLDER_ID — Drive folder for PDF mirror
  • envSUBMISSION_SENDER — DWD impersonation user (eukrit@goco.bz)
  • envSLACK_SIGNING_SECRET_NAME — Secret Manager key for signature verification
  • SMslack-bot-token — Slack bot token (chat.postMessage, modal open)
  • SMslack-signing-secret — verifies inbound interactivity payloads
  • DWDscopes: gmail.send, gmail.modify, gmail.labels, drive.file
